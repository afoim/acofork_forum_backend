## 开发规范

### Git 提交规范

每次完成修改后，需要进行 Git 提交。提交时请遵循以下规范：

1. **每次完成一个功能/修改后立即提交**
2. **提交到本地仓库，不要推送到远程仓库**（除非用户明确要求）
3. **使用语义化提交信息**，格式如下：
   - `feat: 添加新功能`
   - `fix: 修复 bug`
   - `refactor: 重构代码`
   - `docs: 文档更新`
   - `style: 代码格式调整`
   - `perf: 性能优化`
   - `test: 测试相关`
   - `chore: 构建/工具相关`

### 示例

```bash
git add .
git commit -m "feat: 将 WebSocket 实时推送迁移为 SSE"
```
